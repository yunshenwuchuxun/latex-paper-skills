# peft-survey-2022-2026

English | [简体中文](README.zh-CN.md)

Review-paper showcase for the `paper-from-zero` + `arxiv-paper-writer` workflow.

## What this example demonstrates

This project is the review-route example retained in the public repository.

It shows the parts of the workflow that matter for survey writing:

- topic framing and scope definition
- plan creation before prose
- issues-driven drafting and verification
- literature-note organization alongside the paper source
- final review-paper compilation into `main.pdf`

## What this project keeps

- `main.tex` and `main.pdf`
- `plan/` and `issues/`
- `ref.bib`
- `notes/literature-notes.md`

## Start here

If you want to inspect this example quickly, open these in order:

1. `main.pdf` — the final survey artifact.
2. `plan/` — the planning output before full drafting.
3. `issues/` — the execution contract for the writing loop.
4. `notes/literature-notes.md` — the lightweight evidence trail.
5. `main.tex` — the final structured review manuscript.

## Why it is a useful showcase

- It shows the review route: topic framing, literature organization, plan approval, issues-driven writing, citation verification, and final PDF output.
- Unlike the empirical showcase, this project does not depend on a runnable dataset/experiment harness; its evidence trail is the literature and the writing contract.
